# EventCardTikTok — Концепция редизайна

## 🎯 Дизайн-философия

**"Full-screen, immersive, action-first"** — карточка, которая заполняет экран и фокусирует внимание на ключевом действии пользователя.

### Ключевые принципы:
1. **Вертикальная иерархия** — информация течет сверху вниз, от самого важного к дополнительному
2. **Контраст и фокус** — CTA (кнопка регистрации) должна быть визуально доминирующей
3. **TikTok-native паттерны** — большой текст, свободное пространство, минималистичные элементы управления
4. **Социальное доказательство** — участники и группа усиливают вовлеченность

---

## 📐 Структура карточки (вертикальный поток)

### Зона 1: Header (Верхняя четверть экрана ~20-25vh)
**Назначение:** Контекст и навигация

```
┌─────────────────────────────────┐
│ [Аватар] Название группы    [Share] │
│                                   │
│                                   │
│ Заголовок события                │
│ (крупный, жирный, 2-3 строки)   │
└─────────────────────────────────┘
```

**Изменения:**
- Группа и кнопка Share остаются компактными в верхней части
- **Заголовок события значительно увеличивается** (28-32px на мобильных, line-height 1.2)
- Увеличивается отступ между группой и заголовком (24px вместо 16px)
- Заголовок занимает 2-3 строки максимум, затем обрезается с многоточием

**Обоснование:** Заголовок — это первое, что видит пользователь. В TikTok-формате крупный текст = больше внимания и вовлеченности.

---

### Зона 2: Body (Центральная часть ~35-40vh)
**Назначение:** Основной контент и ключевая информация

```
┌─────────────────────────────────┐
│                                   │
│ Описание (читаемый размер)       │
│ ...                               │
│                                   │
│ ┌─────────────────────────────┐ │
│ │ 📅 Дата и время              │ │
│ └─────────────────────────────┘ │
│                                   │
│ ┌─────────────────────────────┐ │
│ │ 📍 Место                     │ │
│ └─────────────────────────────┘ │
│                                   │
└─────────────────────────────────┘
```

**Изменения:**
- Описание остается читаемым (15-16px), но получает больше вертикального пространства
- **Время и место объединяются в компактные карточки** (pill-shaped badges) вместо отдельных блоков
- Карточки располагаются вертикально с отступами 12px
- Используется мягкий фон `rgba(255, 255, 255, 0.85)` для карточек
- Информация о количестве участников **убирается из этого блока** (перемещается в footer)

**Обоснование:** 
- Компактные карточки занимают меньше места и выглядят современнее
- Вертикальное расположение улучшает сканирование
- Убираем дублирование информации (участники показываем только в footer)

---

### Зона 3: Footer (Нижняя треть ~30-35vh)
**Назначение:** Действие и социальное доказательство

```
┌─────────────────────────────────┐
│                                   │
│ [Орбиты участников - центр]      │
│                                   │
│ ┌─────────────────────────────┐ │
│ │    🎉 ЗАПИСАТЬСЯ            │ │ ← ДОМИНИРУЮЩАЯ КНОПКА
│ └─────────────────────────────┘ │
│                                   │
└─────────────────────────────────┘
```

**Изменения:**
- **Кнопка регистрации становится визуально доминирующей:**
  - Размер: padding 18px 24px (больше, чем сейчас)
  - Шрифт: 17-18px, weight: 700 (bold)
  - Иконка: 22px (больше)
  - Фон: более яркий и контрастный (`rgba(255, 255, 255, 0.95)` или цветная версия для зарегистрированных)
  - Скругление: 20px (более мягкое)
  - Тень: subtle shadow для глубины
- **Орбиты участников перемещаются НАД кнопкой** (визуальная иерархия: сначала социальное доказательство, потом действие)
- Отступ между орбитами и кнопкой: 20px
- Кнопка находится внизу с отступом 24px от края экрана (safe-area aware)

**Обоснование:**
- Кнопка регистрации — это главное действие пользователя. Она должна быть настолько заметной, что пользователь не может её пропустить
- Размещение орбит над кнопкой создает визуальный путь: "Смотри, кто уже записался → Теперь твоя очередь"

## 🎨 Визуальные улучшения

### Типографика

| Элемент | Текущий | Предлагаемый | Обоснование |
|---------|---------|--------------|-------------|
| Заголовок события | 24px, weight 800 | **28-32px**, weight 800 | Больше визуальное воздействие, лучше читаемость на расстоянии |
| Описание | 15px, weight 400 | 15-16px, weight 400 | Остается читаемым, но можно немного увеличить |
| Кнопка регистрации | 15px, weight 600 | **17-18px, weight 700** | Кнопка должна быть заметной, крупный текст = больше доверия |
| Метаданные (время/место) | 14px, weight 500 | 14px, weight 500 | Компактно, но читаемо |

### Цвета и контраст

**Кнопка регистрации (не зарегистрирован):**
- Фон: `rgba(255, 255, 255, 0.95)` или `rgba(255, 255, 255, 1)` с легкой тенью
- Текст: `#1F2A1F` (темный, высокий контраст)
- Тень: `0 4px 12px rgba(0, 0, 0, 0.1)`

**Кнопка регистрации (зарегистрирован):**
- Фон: `rgba(92, 194, 170, 0.25)` (более насыщенный, чем сейчас)
- Текст: `rgb(34, 197, 94)` (яркий зеленый)
- Иконка: того же цвета
- Тень: `0 4px 12px rgba(92, 194, 170, 0.2)`

**Карточки метаданных (время/место):**
- Фон: `rgba(255, 255, 255, 0.85)`
- Текст: `#2f2d32`
- Граница: `rgba(0, 0, 0, 0.05)`
- Скругление: 16px

### Пространство и отступы

**Увеличенные отступы между блоками:**
- Header → Body: 24px (вместо 0)
- Body → Footer: 32px (вместо 20px)
- Карточки метаданных: gap 12px (вертикально)
- Footer → Кнопка: 24px (снизу, safe-area aware)

**Центрирование:**
- Кнопка регистрации: `margin: 0 auto` (если нужна фиксированная ширина, например, 90% max-width)
- Орбиты участников: уже центрированы

## 📱 Использование пространства экрана

### Вертикальное распределение (100vh):

```
┌─────────────────────┐
│ Header: ~20-25%     │ ← Компактный, но информативный
├─────────────────────┤
│ Body: ~35-40%       │ ← Основной контент и метаданные
├─────────────────────┤
│ Footer: ~30-35%     │ ← Действие и социальное доказательство
└─────────────────────┘
```

### Принцип "скролла вниз":
Если контент не помещается, пользователь может немного проскроллить вниз, но основные элементы (заголовок, кнопка) остаются видимыми.

## ✨ Ключевые дизайнерские решения

### 1. Увеличение заголовка события
**Почему:** В TikTok-формате крупный текст = больше эмоционального воздействия. Пользователь должен сразу понять, о чем мероприятие.

**Как:** 28-32px на мобильных, line-height 1.2, максимум 3 строки с многоточием.

---

### 2. Компактные карточки для метаданных
**Почему:** Отдельные блоки для времени и места занимают много места. Pill-shaped карточки современнее и компактнее.

**Как:** 
- Вертикальное расположение
- Скругление 16px
- Фон `rgba(255, 255, 255, 0.85)`
- Отступы 12px между карточками

---

### 3. Доминирующая кнопка регистрации
**Почему:** Кнопка регистрации — это главное действие. Она должна быть настолько заметной, что пользователь не может её пропустить.

**Как:**
- Увеличенный размер (padding 18px 24px)
- Крупный текст (17-18px, weight 700)
- Яркий фон и контраст
- Тень для глубины
- Размещение внизу экрана (всегда видна при скролле)

---

### 4. Орбиты участников над кнопкой
**Почему:** Социальное доказательство усиливает желание записаться. Визуальный путь: "Смотри, кто уже записался → Теперь твоя очередь".

**Как:** Орбиты центрированы, отступ 20px над кнопкой.

---

### 5. Убрана информация о количестве участников из метаданных
**Почему:** Дублирование информации. Орбиты уже показывают участников визуально.

**Как:** Удаляем блок `.participantsInfo` из `.importantInfo`.

---

## 🎬 Анимации и микро-взаимодействия

### 1. Анимация появления карточки
**Текущая:** Мягкое появление с масштабированием
**Улучшение:** Сохранить, но добавить небольшую задержку для заголовка (stagger effect)

```typescript
// Заголовок появляется первым, затем описание, затем footer
header: { delay: 0.1 }
body: { delay: 0.2 }
footer: { delay: 0.3 }
```

---

### 2. Интерактивность кнопки регистрации
**Текущая:** `scale(0.98)` при active
**Улучшение:**
- Hover (desktop): легкий подъем `translateY(-2px)`, увеличение тени
- Active: `scale(0.97)` (более выраженный)
- Success state: более яркая анимация (пульсация, confetti уже есть)

---

### 3. Микро-анимация карточек метаданных
**Новое:** При появлении карточки слегка "выпрыгивают" вверх (subtle bounce)

```css
@keyframes cardSlideIn {
  0% { opacity: 0; transform: translateY(10px); }
  100% { opacity: 1; transform: translateY(0); }
}
```

---

### 4. Плавное появление орбит участников
**Новое:** Орбиты появляются с задержкой после загрузки данных, создавая эффект "оживления" карточки

## 📝 Дополнительные замечания

### Адаптивность:
- На экранах <360px: заголовок 26px (вместо 28-32px)
- Кнопка регистрации: минимальная ширина 280px (для удобства клика)
- Карточки метаданных: могут быть в одну колонку на очень маленьких экранах

### Доступность:
- Увеличить область клика для кнопки регистрации (минимум 44x44px)
- Сохранить достаточный цветовой контраст для текста (WCAG AA минимум)
- Добавить aria-labels для интерактивных элементов

### Производительность:
- Сохранить текущие оптимизации (lazy loading, мемоизация)
- Микро-анимации должны быть GPU-ускоренными (`transform`, `opacity`)
