# Оптимизация производительности для мобильных устройств

## Проблема

В мобильной версии контент из вкладок загружался медленно из-за:

- Полной перезагрузки компонентов при переключении вкладок
- Отсутствия предзагрузки соседних компонентов
- Неэффективного кэширования
- Отсутствия адаптации под производительность устройства

## Решения

### 1. Умное кэширование компонентов (`SmartCache`)

- **Временное кэширование**: компоненты кэшируются на 5 минут
- **Ограничение размера**: максимум 10 компонентов в кэше
- **Автоочистка**: старые компоненты удаляются автоматически

### 2. Адаптивная предзагрузка

- **Детекция производительности**: определяет мощность устройства
- **Анализ соединения**: учитывает скорость интернета
- **Умная стратегия**: загружает разное количество компонентов в зависимости от условий

### 3. Оптимизированный рендеринг вкладок

- **Избежание ремаунта**: компоненты остаются в DOM, но скрываются
- **Постепенная загрузка**: соседние вкладки загружаются в фоне
- **Приоритизация**: активная вкладка всегда загружается первой

### 4. Производительные настройки

- **Отключение анимаций**: на слабых устройствах
- **Debounce для поиска**: предотвращает частые запросы
- **Мемоизация**: предотвращает лишние ререндеры

## Технические детали

### Детекция производительности

```typescript
// Определяем производительность по:
- navigator.hardwareConcurrency (количество ядер)
- navigator.deviceMemory (объем памяти)
- connection.effectiveType (скорость соединения)
- prefers-reduced-motion (предпочтения пользователя)
```

### Стратегии загрузки

- **Высокая производительность**: предзагрузка 3 компонентов с задержкой 500мс
- **Средняя производительность**: предзагрузка 2 компонентов с задержкой 1500мс
- **Низкая производительность**: предзагрузка 1 компонента с задержкой 2000мс

### Кэширование поиска

- Поисковые запросы сохраняются для каждой вкладки отдельно
- Восстанавливаются при возврате к вкладке
- Debounce 300мс для localStorage операций

## Результаты оптимизации

### Ожидаемые улучшения:

- **Скорость переключения вкладок**: до 70% быстрее
- **Потребление памяти**: умное кэширование предотвращает утечки
- **Батарея**: меньше CPU нагрузки на слабых устройствах
- **Пользовательский опыт**: более плавные переходы

### Мобильные оптимизации:

- Адаптивные размеры элементов для маленьких экранов
- Упрощенные анимации на слабых устройствах
- Оптимизированные media queries для различных разрешений

## Файлы изменений

- `page.tsx` - основной компонент с оптимизированным рендерингом
- `useHomepageOptimization.tsx` - улучшенный хук состояния
- `useMobileOptimization.tsx` - новый хук детекции производительности
- `page.module.scss` - адаптивные стили и оптимизации

## Совместимость

Все оптимизации работают с fallback'ами для старых браузеров и не ломают существующую функциональность.
